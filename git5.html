Git Flow전략 (by Vincent Driessen)



-main => 유저 배포용 (최종 결과물)
-develop => 개발용
-feature/guild, feature/friend => 신기능 -release => 배포전 테스트 단계

-유저 배포후 오류 발생 시 - hot fix branch 생성 해서 바로 고치기
-release,hot fix는 develop에도 merge해주길

Trunk -Based전략(브랜치 하나만 잘 관리하자)



- main 브랜치에 곁가지 브랜치를 그때그때 붙임
- 소스 코드 한곳에만 있어서 편함 (메인만 관리하면 됨)
- 하지만 테스트 많이 자주 해야 됨
- 안정화된 프로젝트에 많이 쓰임
