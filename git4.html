1. 코드 쓰다가 망하면? 파일 복사본 만들어서 코드 짜보기(테스트용)
1_1. branch기능을 쓰면 복사본 만들기 쉬움 (branch => commit의 복사본 느낌)
2. git branch 브랜치명 (branch생성 => 기존 파일의 복사본을 만들어서 안전하게 먼저 작업 해보고 적용하자)(최근 파일의 사본)
3. git swich 브랜치명 (브랜치로 이동)
4. 브랜치든 메인이든 add랑 commit 해놓자
5. git merge 브랜치명 (브랜치와 메인 합치기)
5_1. merge 할때 같은 파일의 같은 부분이 둘다 다르게 수정 되어있을 경우 충돌(config) 발생 충돌 해결하고 add, commit 하자
